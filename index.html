Framework ágil iterativo e incremental para entregas frequentes de valor

Introdução

Scrum é um framework ágil iterativo e incremental que promove entregas frequentes de valor, transparência, inspeção e adaptação. No PDS Corporativo, o Scrum é utilizado para garantir planejamento colaborativo, previsibilidade, qualidade técnica e evolução contínua do produto.

Este documento descreve o processo Scrum adotado, incluindo o fluxo de trabalho, papéis, atividades, práticas e políticas.

Detalhamento do Fluxo

  1. Caso de uso é avaliado pelo grupo
  2. Caso de uso gera uma User Story
  3. A Story recebe:
    • Prioridade (negócio)
    • Pontos (baseados no esforço total)
  4. A Story é quebrada em Tasks
  5. Cada Task recebe uma estimativa de tempo
  6. A soma das Tasks define a pontuação final da Story
  7. As Stories priorizadas entram na Sprint
  8. A Sprint é executada, testada e revisada

Políticas do Fluxo

  • Stories sempre refinadas antes da Sprint
  • Pontuação baseada no esforço total das tasks
  • Sprint com escopo fechado
  • DoR e DoD explícitos
  • Feedback contínuo via Review e Retro

Matriz RACI - Associação de Papéis às Atividades

A matriz RACI define a responsabilidade de cada papel em cada atividade do processo:

  • R (Responsible) - Responsável pela execução
  • A (Accountable) - Autoridade e aprovador final
  • C (Consulted) - Consultado para decisões
  • I (Informed) - Informado sobre o andamento

Práticas Ágeis

Práticas Gerais

  • Scrum Events (Planning, Daily, Review, Retro)
  • Sprint fixa (ex: 2 semanas)
  • Definition of Ready (DoR)
  • Definition of Done (DoD)
  • Estimativa por Task
  • Pontuação da Story baseada no esforço total
  • Testes automatizados
  • PR pequeno + code review
  • Métricas de velocidade e previsibilidade
  • Melhoria contínua
